# Release FAQ

## Was bewirkt ein GitHub Release?

Ein Release ist **mehr als ein normaler Commit**:

1. **Meilenstein sichtbar machen**  
   Ein Tag + Release-Notes markieren einen stabilen Stand (z. B. `v0.1-alpha`).

2. **Bessere Auffindbarkeit**  
   Viele Nutzer filtern aktiv nach Projekten mit Releases, weil sie Vertrauen und Reife signalisieren.

3. **Teilbar wie ein Produkt-Update**  
   Du kannst auf X/LinkedIn/Discord direkt auf eine konkrete Version verlinken.

4. **Onboarding vereinfachen**  
   Neue Nutzer wissen sofort: „Was ist neu? Was ist bekannt? Wie starte ich?“

5. **Grundlage für Ökosystem-Tools**  
   Changelogs, Package-Pipelines und manche Bots orientieren sich an Tags/Releases.

## Bedeutet ein Release automatisch mehr Publicity?

**Nicht automatisch.**  
Ein Release ist ein **starker Multiplikator**, aber keine Werbung von allein.

Publicity entsteht vor allem durch:
- Posting/Ankündigung (Social, Communities, Newsletter),
- gute Demo/Visuals,
- klare Release Notes,
- niedrige Einstiegshürde.

## Empfehlung für Mom4AI Forge

Ein `v0.1-alpha` Release macht Sinn als „Einladung zum Mitbauen“.  
Für Reichweite: Release + kurze Demo + klarer CTA („Teste `MomAI`, teile dein bestes Skelett“).

# Mom4AI Forge v0.1.0-alpha – Release Notes (Draft)

Erste öffentliche Alpha-Version der evolutiven Mom4AI Forge.

## Highlights

- Bio-inspirierte Skelett-Generierung (Graph/DAG)
- Auto-Fitness + Resonance Protocol v0.1
- Hall-of-Fame mit Resonanz-Kennzahlen
- Lokaler Hybrid-Dashboard-Server
- 1-Kommando-CLI: `MomAI`
- Baseline CI + MIT License

## Enthaltene Features

1. **Skelett-Fabrik**
   - Erzeugung, Bewertung, Speicherung und Visualisierung von Architektur-Skeletten.
2. **Resonanz als Überlebenssignal**
   - JSONL-basierte Interaktionssignale als zweites Selektionskriterium.
3. **Local Runtime Experience**
   - Web-Dashboard + lokale API-Endpunkte für Status/Stats.
4. **Projekt-Reife**
   - Packaging-Grundlage, Changelog, Release-Readiness-Checkliste.

## Bekannte Grenzen in Alpha

- Mutation/Crossover noch nicht vollständig als produktive Linie integriert.
- „Skelett → trainierbares Child-GPT“ ist noch Roadmap.
- Kein fertiges Login-/Katalog-Portal für Enduser-Model-Browsing.

## Install / Start

```bash
pip install -r requirements.txt
pip install -e .
MomAI
```

Lokal-Dashboard:

```bash
python src/live_dashboard_server.py
```

## Upgrade-Hinweis

Wenn du vor der CLI-Einführung bereits lokal arbeitest:

```bash
git pull
pip install -e .
```

Dann kannst du statt `python src/mom_forge.py` künftig direkt `MomAI` nutzen.
